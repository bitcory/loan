**Step A: src/lib/customer-number.ts 수정**

현재 파일을 읽어 `generateCustomerNumber` 함수의 시그니처를 다음으로 업데이트한다:

```typescript
import { getTenantClient } from "@/lib/prisma";

type TenantDb = ReturnType<typeof getTenantClient>;

export async function generateCustomerNumber(db: TenantDb): Promise<string> {
  const last = await db.customer.findFirst({
    orderBy: { customerNumber: "desc" },
    select: { customerNumber: true },
  });
  const next = last
    ? parseInt(last.customerNumber.replace("C-", ""), 10) + 1
    : 1;
  return `C-${String(next).padStart(4, "0")}`;
}
```

기존에 `prisma`를 import해서 직접 사용하던 코드를 `db` 파라미터로 교체한다. 조직별 독립 채번이므로 ctx.db (테넌트 격리 클라이언트)가 반드시 전달되어야 한다.

조회 함수(getCustomers, getCustomer 등)는 Server Component에서 직접 호출되므로:
- `getServerSession(authOptions)` + `getTenantClient(session.user.organizationId)` 패턴 사용

- `authenticatedAction` (조회/생성/수납) 또는 `adminAction` (삭제) 으로 래핑

   ```typescript
   export const updateSetting = adminAction
     .schema(z.object({ key: z.string(), value: z.string() }))
     .action(async ({ parsedInput, ctx }) => {
       await ctx.db.setting.update({
         where: { organizationId_key: { organizationId: ctx.organizationId, key: parsedInput.key } },
         data: { value: parsedInput.value },
       });
       revalidatePath("/settings");
       return { success: true };
     });
   ```
