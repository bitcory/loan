schema.prisma 파일 끝(Setting 모델 다음)에 AuditLog 모델을 추가한다:

```prisma
model AuditLog {
  id             String   @id @default(cuid())
  organizationId String
  userId         String
  entityType     String   // "Customer" | "Loan" | "Collateral" | "Mortgage" | "Payment" | "Setting"
  entityId       String
  action         String   // "CREATE" | "UPDATE" | "DELETE"
  before         Json?    // CREATE의 경우 null
  after          Json?    // DELETE의 경우 null
  ipAddress      String?
  userAgent      String?
  createdAt      DateTime @default(now())

  @@index([organizationId, createdAt])
  @@index([organizationId, entityType, entityId])
  @@index([organizationId, userId])
  @@map("audit_logs")
}
```

설계 근거 (변경하지 말 것):
- updatedAt 필드 없음 — append-only 레코드는 업데이트되지 않는다
- Organization/User와 @relation 없음 — FK cascade가 로그를 삭제할 수 없도록 의도적으로 제거
- organizationId는 plain String — basePrisma로 직접 쓰기, 테넌트 extension 불포함

Task 1에서 생성된 migration.sql 파일 끝에 다음 두 줄을 추가한다:

```sql
-- Append-only enforcement: silently drop UPDATE and DELETE at DB level
CREATE RULE no_update_audit_logs AS ON UPDATE TO audit_logs DO INSTEAD NOTHING;
CREATE RULE no_delete_audit_logs AS ON DELETE TO audit_logs DO INSTEAD NOTHING;
```

주의사항:
- Prisma가 자동 생성한 CREATE TABLE 문 이후에 추가한다
- 기존 생성된 SQL은 변경하지 않는다
- RULE은 DO INSTEAD NOTHING이므로 실행해도 에러 없이 0 rows affected를 반환한다

SQL 추가 후 마이그레이션을 적용한다:
```bash
npx prisma migrate deploy
```

마이그레이션 성공 후 Prisma Client를 재생성한다:
```bash
npx prisma generate
```

append-only 동작 검증 (psql 또는 npx prisma db execute로 실행):
```sql
-- 테스트용 row 삽입
INSERT INTO audit_logs (id, "organizationId", "userId", "entityType", "entityId", action, "createdAt")
VALUES ('test-id', 'org-1', 'user-1', 'Customer', 'cust-1', 'CREATE', NOW());

-- UPDATE 시도 — 0 rows affected여야 함
UPDATE audit_logs SET action = 'DELETE' WHERE id = 'test-id';

-- DELETE 시도 — 0 rows affected여야 함
DELETE FROM audit_logs WHERE id = 'test-id';

-- 원본 레코드 여전히 존재 확인
SELECT action FROM audit_logs WHERE id = 'test-id';
-- 결과: 'CREATE' (변경되지 않음)

-- 테스트 레코드 정리는 Rule이 막으므로 아래로 우회 (개발 환경에서만):
-- TRUNCATE audit_logs;
```
