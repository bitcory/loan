**createCustomer** (현재 구조 유지, logAudit 추가):
- customer.create 성공 후, revalidatePath 전에 logAudit 호출
- CREATE의 before는 null
- after: parsedInput에서 residentNumber를 plain-text로 전달 (maskPii가 내부에서 처리)
- 주의: encrypted 값이 아닌 parsedInput.residentNumber (원문)를 after에 사용
```typescript
await logAudit(
  { userId: ctx.userId, organizationId: ctx.organizationId, ipAddress: getClientIp() },
  "Customer",
  customer.id,
  "CREATE",
  null,
  sanitizeForLog({ ...rest, customerNumber, residentNumber: residentNumber ?? null }),
);
```

**updateCustomer** (before 스냅샷 추가 필요):
- update 실행 전에 `const existing = await ctx.db.customer.findFirst({ where: { id: parsedInput.id } });` 추가
- update 성공 후, revalidatePath 전에 logAudit 호출
- before: existing을 sanitizeForLog로 변환 (null-safe)
- after: parsedInput.data에서 residentNumber plain-text 사용
```typescript
const existing = await ctx.db.customer.findFirst({ where: { id: parsedInput.id } });
// ... existing update logic ...
await logAudit(
  { userId: ctx.userId, organizationId: ctx.organizationId, ipAddress: getClientIp() },
  "Customer",
  parsedInput.id,
  "UPDATE",
  existing ? sanitizeForLog(existing as unknown as Record<string, unknown>) : null,
  sanitizeForLog({ ...rest, residentNumber: residentNumber ?? null }),
);
```

**deleteCustomer** (before 스냅샷 추가 필요):
- 활성 대출 체크(기존 코드) 이후, delete 이전에 `const existing = await ctx.db.customer.findFirst({ where: { id: parsedInput.id } });` 추가
- delete 성공 후, revalidatePath 전에 logAudit 호출
```typescript
await logAudit(
  { userId: ctx.userId, organizationId: ctx.organizationId, ipAddress: getClientIp() },
  "Customer",
  parsedInput.id,
  "DELETE",
  existing ? sanitizeForLog(existing as unknown as Record<string, unknown>) : null,
  null,
);
```

주의사항:
- `existing`을 `ctx.db.customer.findFirst`로 가져올 때 `residentNumber`는 암호화된 상태다. sanitizeForLog가 maskPii를 호출하므로 "***MASKED***"로 저장된다 — 이것이 올바른 동작이다.
- logAudit는 try/catch 내부에서 에러를 삼키므로 에러 전파를 걱정할 필요 없다.
